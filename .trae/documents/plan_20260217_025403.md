## 问题总结

### 根本原因
1. **设备绑定关系不一致**：模拟器发送 `DRONE_GATEWAY_001` 的 OSD 数据，但 Redis/数据库中 GATEWAYSIM001 的子设备是 `DRONE_SIM001`
2. **DRONE_GATEWAY_001 的 domain 错误**：数据库中 domain=1 (PAYLOAD)，应该是 domain=0 (DRONE)

### 解决方案

**方案 A：统一使用 DRONE_SIM001（推荐）**
1. 修改 `gateway_simulator.py` 中的 `DRONE_SN = "DRONE_SIM001"`
2. 模拟器发送 `thing/product/DRONE_SIM001/osd` 数据

**方案 B：统一使用 DRONE_GATEWAY_001**
1. 修改数据库：`UPDATE manage_device SET child_sn='DRONE_GATEWAY_001' WHERE device_sn='GATEWAYSIM001'`
2. 修改数据库：`UPDATE manage_device SET domain=0 WHERE device_sn='DRONE_GATEWAY_001'`
3. 重启后端服务让 Redis 重新加载

### 推荐方案 A
因为 `DRONE_SIM001` 已经在系统中正确配置（domain=0，绑定关系正确），只需修改模拟器配置即可。
