## 问题分析

### 根本原因
数据库中 `DOCKSIM001` 和 `GATEWAYSIM001` 都设置了 `child_sn = 'DRONE_SIM001'`，导致：
1. DRONE_SIM001 被注册到 DOCKSIM001
2. 无人机 OSD 数据的 `gateway` 字段变成 `DOCKSIM001`
3. GATEWAYSIM001 无法获取无人机数据

### 解决方案

**方案 1：为 GATEWAYSIM001 创建独立的无人机设备**
```sql
-- 创建新的无人机设备
INSERT INTO manage_device (device_sn, device_name, nickname, workspace_id, device_type, sub_type, domain, bound_status, create_time, update_time)
VALUES ('DRONE_GATEWAY_001', 'DJI M30T', 'Gateway Drone', 'e3dea0f5-37f2-4d79-ae58-490af3228069', 89, 0, 1, 1, UNIX_TIMESTAMP()*1000, UNIX_TIMESTAMP()*1000);

-- 更新 GATEWAYSIM001 的子设备
UPDATE manage_device SET child_sn = 'DRONE_GATEWAY_001' WHERE device_sn = 'GATEWAYSIM001';

-- 恢复 DOCKSIM001 的原始子设备（如果有）
-- 或者保持 DRONE_SIM001 给 DOCKSIM001
```

**方案 2：修改模拟器**
- 更新模拟器发送无人机 OSD 时使用正确的 gateway 字段

### 推荐方案
采用方案 1，为 GATEWAYSIM001 创建独立的无人机设备，同时更新模拟器发送到正确的 Topic。
