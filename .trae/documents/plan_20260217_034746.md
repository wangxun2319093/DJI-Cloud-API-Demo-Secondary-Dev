## 问题总结

### 根本原因
数据库中 `GATEWAYSIM001.child_sn` 被改回了 `DRONE_SIM001`，但模拟器发送的 status 消息中子设备是 `DRONE_GATEWAY_001`。

### 解决方案

**方案 A：统一使用 DRONE_GATEWAY_001**
1. 修改数据库：`UPDATE manage_device SET child_sn='DRONE_GATEWAY_001' WHERE device_sn='GATEWAYSIM001'`
2. 清除 Redis 缓存：`DEL online:GATEWAYSIM001 online:DRONE_SIM001`
3. 重启模拟器，重新发送 status 消息

**方案 B：统一使用 DRONE_SIM001**
1. 修改模拟器：`DRONE_SN = "DRONE_SIM001"`
2. 重启模拟器

### 推荐方案 A
因为模拟器已经配置为 `DRONE_GATEWAY_001`，只需修改数据库即可。
