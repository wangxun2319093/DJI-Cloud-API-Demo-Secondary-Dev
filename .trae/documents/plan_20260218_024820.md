## 问题分析

### EMQX 收到的数据

**无人机 OSD** (DRONESIM001):
```json
{
  "gateway": "DOCKSIM001",  // ❌ 问题在这里！
  "data": {
    "height": 74.4,
    "horizontalSpeed": 12.8,
    "homeDistance": 195.91,
    ...
  }
}
```

**Gateway OSD** (GATEWAYSIM001):
```json
{
  "gateway": "GATEWAYSIM001",
  "data": {
    "latitude": 30.5728,
    "capacity_percent": 99,
    "wireless_link": {...}
  }
}
```

### 问题根因

模拟器发送无人机 OSD 时，`gateway` 字段始终是 `DOCKSIM001`，导致：

1. 后端将无人机 OSD 路由到 `OsdDockDrone` 处理器
2. 前端 Online Devices 面板期望无人机 OSD 的 `gateway` 是 `GATEWAYSIM001`
3. 数据关联错误，前端无法显示无人机详细数据

### 解决方案

修改模拟器，为 Gateway 组合单独发送一份无人机 OSD 数据：

1. **新增 `_publish_gateway_drone_osd()` 方法**
   - 发送 gateway 字段为 `GATEWAYSIM001` 的无人机 OSD
   - 这样后端会路由到 `OsdRcDrone` 处理器
   - 前端能正确关联 Gateway 和 Drone 数据

2. **修改 `_publish_osd()` 方法**
   - 在发送 Dock 相关 OSD 后，额外发送 Gateway 相关的无人机 OSD
