## 问题总结

### 根本原因
Redis 缓存未更新，数据库修改后 Redis 仍保留旧的绑定关系：
- 数据库：`GATEWAYSIM001.child_sn = "DRONE_GATEWAY_001"`
- Redis：`GATEWAYSIM001.child_device_sn = "DRONE_SIM001"`

### 解决方案

**方案 A：清除 Redis 缓存重新注册（推荐）**
1. 删除 `online:GATEWAYSIM001` 缓存
2. 让模拟器重新发送 status 消息
3. 后端重新从数据库加载绑定关系

**方案 B：直接更新 Redis 缓存**
1. 更新 `online:GATEWAYSIM001` 中的 `child_device_sn` 为 `DRONE_GATEWAY_001`
2. 添加 `online:DRONE_GATEWAY_001` 在线状态

### 执行步骤
1. 删除 Redis 中的 `online:GATEWAYSIM001` 缓存
2. 让模拟器重新发送 status 消息注册设备
3. 验证前端展示
