## 修复计划

### 问题原因
后端重启后，WebSocket 会话的本地缓存 `SESSIONS` 被清空，导致无法向前端推送消息。

### 修复步骤

#### 1. 清理 Redis 中的旧 WebSocket 会话数据
```bash
redis-cli -h 118.195.150.246 -p 6004 -n 2 DEL "webSocket:e3dea0f5-37f2-4d79-ae58-490af3228069"
redis-cli -h 118.195.150.246 -p 6004 -n 2 DEL "webSocket:Web"
```

#### 2. 前端重新连接
- 刷新前端页面，让 WebSocket 重新建立连接
- 新连接会重新注册到 Redis 和本地 SESSIONS

#### 3. 验证设备数据推送
- 确认模拟器发送 status 消息
- 后端处理并推送 DEVICE_ONLINE 到前端
- 前端收到消息并展示设备

### 可选优化（长期）
修改 `WebSocketManageServiceImpl`，在获取会话时如果发现 Redis 中有 sessionId 但本地 SESSIONS 中没有，自动清理 Redis 中的无效数据。
