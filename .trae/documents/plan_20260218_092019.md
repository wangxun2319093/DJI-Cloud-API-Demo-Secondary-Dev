## 问题分析

### 数据流程

1. **Status 消息** → 后端注册设备 → 订阅相关 topic
2. **OSD 消息** → 后端根据 `gateway` 字段路由 → 推送到 WebSocket → 前端显示

### 根本问题

后端没有收到 status 消息，导致 GATEWAYSIM001 没有在 SDK 中注册。后端收到 OSD 消息后，无法找到 GATEWAYSIM001 的类型，导致消息被路由到 default channel。

### 解决方案

**重启后端服务，确保 MQTT 订阅正确生效：**

1. 停止当前后端服务
2. 清空 Redis 数据
3. 重启后端服务
4. 重启模拟器
5. 验证 status 消息被正确接收
6. 验证 GATEWAYSIM001 在 SDK 中注册
7. 验证前端 Online Devices 显示无人机数据

### 预期结果

* 后端收到 status 消息

* GATEWAYSIM001 在 SDK 中注册为 RC 类型

* Gateway Drone OSD 被正确路由到 `OsdRcDrone` 处理器

* 前端 Online Devices 面板显示无人机数据
